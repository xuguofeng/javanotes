一、面向对象思想

面向对象编程是当今主流的程序设计思想，已经取代了过程化程序开发技术，java 是完全面向对象编程语言，必须熟悉面向对象才能够编写 java 程序。

面向对象的程序是由对象组成的，每个对象包含着对用户公开的特定功能和隐藏的实现部分。程序中的很多对象来自标准库，还有一些需要自定义。

从根本上说，只要对象能够实现业务功能，其具体的实现细节不必特别关心。

面向对象有以下特点：

（1）面向对象是一种常见的思想，比较符合人们的思考习惯；

（2）面向对象可以将复杂的业务逻辑简单化；

（3）面向对象具有抽象、封装、继承、多态等特性。

面向对象的编程语言主要有：C++、Java、C#等。

二、类和对象的关系

类：

对某类事物的普遍一致性特征、功能的抽象、描述和封装，是构造对象的模版或蓝图，用 java 编写的代码都会在某些类的内部。类之间的关系主要有：依赖、聚合、继承。


对象：

使用 new 关键字创建的类的实例。同一个类的所有对象，都具有相似的数据和行为，但是每个对象都保存着自己独特的状态，对象状态会随着程序的运行而发生改变，需要注意状态的变化必须通过调用方法来改变，这就是封装的基本原则。

三、类和对象举例

每个人都有手机，我们就使用手机来举例。

手机有操作系统、手机号、品牌、电量等重要属性，有发短信、充电等功能。我们就使用 java 代码编写一个手机类。 

public class MobilePhone1 {
	
	// 操作系统
	String os;

	// 手机号码
	String phoneNumber;

	// 手机品牌
	String brand;

	// 剩余电量
	double dumpEnergy;

	// 发短信的方法
	void sendMessage(String message, String targetPhoneNumber){
		System.out.println("发给" + targetPhoneNumber + ", 内容是：" + message);
	}

	// 充电方法
	double charge(){
		System.out.println("正在充电, 剩余电量：" + dumpEnergy * 100 + "%");
		return dumpEnergy;
	}

	public static void main(String[] args) {

		// 使用new关键字调用MobilePhone的默认构造方法创建实例
		MobilePhone1 mobile = new MobilePhone1();

		// 给mobile对象成员变量赋值
		mobile.os = "Andriod";
		mobile.phoneNumber = "12345678901";
		mobile.brand = "红米2";
		mobile.dumpEnergy = 0.87d;

		// 调用mobile对象的方法
		mobile.sendMessage("你好，我是***，这是一个测试短信，打扰了", "12345678902");
		double de = mobile.charge(); // 因为方法有返回值，所以可以使用double类型变量接收

		// 演示匿名对象
		new MobilePhone1().charge();
	}
}

输出：

发给12345678902, 内容是：你好，我是***，这是一个测试短信，打扰了
正在充电, 剩余电量：87.0%
正在充电, 剩余电量：0.0%

有以下几点需要注意：

（1）使用 class 关键字定义类

（2）类名可以是字母、数字、$_的组合，不可以数字开始，首字母大写

（3）.java 源代码文件中 public 的类名称需要和文件名一致

（4）定义类实际上就是定义成员变量和功能方法

（5）成员变量和函数通常使用 public, protect, private 等访问权限关键字进行描述

（6）成员变量在整个类中都有效

（7）使用 new 关键字创建类的实例对象时，会在 JVM 堆内存中开辟空间存储对象

（8）使用赋值运算符 = 可以让变量指向实例对象的存储位置，也就是说变量实际上只保存对象在堆内存的地址值，这就是引用数据类型的本质

（9）匿名对象，没有变量引用的对象。比如下面这段代码： 

// 匿名对象
// 因为成员变量dumpEnergy没有赋值，所以使用初始化值0.0
new MobilePhone().charge();

输出：

正在充电, 剩余电量：0.0%

演示代码在项目 oop 中的 MobilePhone1.java 文件。

四、成员变量和局部变量

（1）成员变量定义在类中，整个类都可以访问此变量，有初始化值，存在于堆内存中，随着对象的创建而创建，随着对象的释放而释放。

（2）局部变量定义在某个代码块中，只在所在代码块有效，没有初始化值，存在于栈内存中，生命周期是所在代码块。

（3）如果某个代码块中的局部变量和成员变量同名，在此代码块访问此变量获取到的是局部变量，如果想获取到成员变量，可以使用 this 关键字来访问对象的成员变量。
