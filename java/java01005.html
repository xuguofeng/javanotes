二、标识符

用于在程序中定义类名、方法名、变量名，由大小写字母、数字、_$组成：

（1）不可以数字开头

（2）不可以使用关键字

（3）严格区分大小写

（4）尽量有意义

三、注释

注释主要用来说明代码意义、调试程序，在 java 中主要有三种注释方式：

第一种注释 

// 单行注释内容

第二种注释 

/*
多行注释内容
*/

第三种注释 

/**
* 文档注释内容，这个内容可以被javadoc命令提取
*/

四、基本数据类型

Java 语言是强类型语言，对于每一种数据都定义了明确的数据类型，从而在内存中分配不同大小的内存空间，主要分为基本数据类型和引用数据类型两大类

在此我们先介绍基本数据类型

五、变量

存储程序运行过程中需要重复使用且变化的数据，拥有自己的名称、类型，值可以在所属类型取值范围内进行变化。命名方式见“标识符”部分。

变量声明后需要在内存中占用一定的存储空间，比如 1 个 int 类型的变量占用 4 个字节，1 个 byte 类型变量占用 1 个字节，而复杂类型变量占用的空间是由其本身的属性来决定的。

变量在一定的作用域内有效，关于这一点将在流程控制、方法、类的讲解中在进行详细介绍。

定义方式如下： 

/*
	第一种方式
*/
类型 变量名;
变量名 = 初始化值;

/*
	第二种方式
*/
类型 变量名 = 初始化值;

/*
	第三种方式
*/
类型 变量名1, 变量名2;
变量名1 =初始化值;
变量名2 =初始化值;

定义 int 类型变量的例子： 

/*
	使用第一种方式声明一个变量a，类型是int，之后为a变量赋值为1
	注意：局部变量必须进行初始化操作，否则无法使用
*/
int a;
a = 1;

/*
	使用第二种方式声明一个变量a，类型是int，赋值为1
*/
int a = 1;

/*
	使用第三种方式声明变量a、b，类型都是int，之后分别赋值为1、2
*/
int a, b;
a = 1;
b = 2;

六、基本数据类型变量定义

下面我们来看一下八种基本数据类型的定义： 

/*
	byte、short、long类型
*/
byte a1 = 1;
short a2 = 2;
long a3 = 3;

System.out.println(a1); // 1
System.out.println(a2); // 2
System.out.println(a3); // 3

/*
	float、double类型
*/
float a4 = 4.67f;
double a5 = 5.78d;
System.out.println(a4); // 4.67
System.out.println(a5); // 5.78

/*
	boolean类型
*/
boolean a6 = true; 
System.out.println(a6); // true

/*
	char类型
*/
char a7 = 'a';
System.out.println(a7); // a

注意整型的取值范围问题： 

/*
	以下定义方式会出现编译错误，因为byte类型的取值范围是[ -128, 127 ]
*/
byte b1 = 200;

七、强制类型转换

为了演示这个问题，我们需要借助 char 和 int ，先看下这段代码 

/*
	使用 (目标类型)变量名 方式来实现类型的强制转换
	如果使用变量来引用，那么该变量类需要和目标类型一致
	这种类型转换的前提是被转换类型可以转为目标类型
*/
char a7 = 'a';
System.out.println(a7); // a

System.out.println( (int)a7 ); // 97 强制类型转换
System.out.println( (byte)a7 ); // 97
System.out.println( (short)a7 ); // 97

int b1 = 101;
System.out.println( (char)b1 ); // e
System.out.println( (char)(b1 + 1) ); // f

System.out.println( (int)('徐') ); // 24464

输出：

97
97
97
e
f
24464

下面这段代码会把 26 个小写字母对应的 int 值打印出来，关于 数组 和 循环结构 我们以后会有详细的讲解 

/*
	char[] chs = {} 用于定义一个 char 类型数组
	for 使用循环结构中的一种，以后会详细讲解
*/
char[] chs = {
			'a','b','c','d','e','f','g','h','i','j','k','l','m',
			'n','o','p','q','r','s','t','u','v','w','x','y','z'};

for (int i = 0; i < chs.length; i++) {
	System.out.println( (int)chs[i] );
}

八、隐式类型转换

关于隐式类型转换的问题我们只会在一些特殊场合遇到，平时的软件开发很少涉及，所以下面的讲解和案例代码以了解为主 

short a = 1;
short b = 2;

short c = a + b; // 出现编译错误，“可能损失精度”

原因在于：在进行 a + b 操作时，会把结果的类型提升为 int，此时使用 short 类型的 c 变量引用时，就会出现这个错误

第一种解决办法：将 a + b 强制类型转换为 short，但是这样的做法可能导致数值精度发生错误 

short a = 1;
short b = 2;

// short c = (a + b);
short c = (short)(a + b);

第二种解决办法：使用 int 或 long 类型的变量引用 a + b 的结果 

short a = 1;
short b = 2;

// short c = (a + b);
// short c = (short)(a + b);
int c = (a + b);
