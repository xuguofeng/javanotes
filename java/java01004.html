一、工作空间与创建项目

工作空间：统一存放 java 项目的目录，我们的全部代码演示项目都在 D:\tmp\java 这个工作空间下；

项目：存放一整套 java 程序源代码（.java文件）和字节码文件（.class文件）的目录。您可以直接把源代码文件放在项目根目录下，也可以创建 src 目录存放源代码文件，创建 bin 目录用于存放字节码文件；

我们就演示下如何在工作空间中创建 java 项目： 

D:\tmp\java>md quickstart

D:\tmp\java>

或者使用界面操作创建文件夹，也是一样的效果。

创建好 quickstart 项目，我们就可以在项目下面编写源代码、或者再创建 src 目录后在 src 目录下编写源代码。

后面我们还会讲解使用包（package）来管理项目中的类。

二、Editplus 开发环境（简单环境）

如果需要经常开发软件，使用 notepad 编写程序不是很方便，所以我们需要更加优秀的文本编辑器来搭建开发环境，Editplus 就是一个很好的选择

下载 editplus 并安装，安装过程略

打开软件，点击“工具”，选择“配置用户工具”

在工具组 1 点击“添加工具”选择应用程序，创建 javac 编译工具，配置参数如下 

菜单文字：javac
命令：D:\Program Files\Java6\jdk1.6.0_45\bin\javac.exe
参数：-d D:\tmp\java\quickstart\. -classpath D:\tmp\java\quickstart\. -encoding UTF-8 $(FileName)
初始目录：$(FileDir)
动作：捕捉输出

菜单文字：自定义，在此我们取名javac
命令：使用的是jdk安装目录下bin下的javac.exe
参数：-d选项指定编译后.class文件的存放位置；
	-classpath选项指定编译时在哪个目录下寻找依赖的类；
	-encoding选项指定使用的编码方式，与源文件的编码一致；
	$(FileName)是一个代表当前文件名称的变量，Editplus会为我们传入。

* 参数中的项目目录需要根据自己的项目确定

使用 editplus 打开之前编写的 QuickStart.java 文件，点击“工具”，可以看到javac工具，点击进行编译，出现编译错误

原来我们之前使用 notepad 编写的 .java 文件使用的 ANSI 编码，而我们在创建 javac 工具时使用的 UTF-8 编码方式，所以需要修改 .java 文件的编码方式。点击“文档”，选择“文件编码”，选择“更改文件编码方式”，将 ANSI 修改为 UTF-8

修改后保存文件，再使用 javac 工具编译

在工具组 1 点击“添加工具”选择应用程序，创建 java 运行工具，配置参数如下 

菜单文字：java
命令：D:\Program Files\Java6\jdk1.6.0_45\bin\java.exe
参数：-classpath D:\tmp\java\quickstart\. $(FileNameNoExt)
初始目录：$(FileDir)
动作：捕捉输出

菜单文字：自定义，在此我们取名java
命令：使用的是jdk安装目录下bin下的java.exe
参数：-classpath选项指定编译时在哪个目录下寻找需要的类。
	$(FileNameNoExt)是不带扩展名的文件名，Editplus会为我们传入。

* 参数中的项目目录需要根据自己的项目确定

使用 java 工具运行 QuickStart

注意：使用这种方式配置较为基础，只带有基础的编译、运行功能，当我们创建了新的项目，又需要修改工具的参数配置，对于初学者比较麻烦，所以我们推荐使用下面的方法进行配置。

三、Editplus 开发环境（高级环境）

解压插件 Editplus4Java.zip ，得到 Editplus4Java.class 文件，将该文件复制到工作空间 D:\tmp\java 目录下，如下：

修改之前配置的 javac 工具 

菜单文字：编译工具
命令：D:\Program Files\Java6\jdk1.6.0_45\bin\java.exe
参数：-classpath D:\tmp\java\. Editplus4Java D:\tmp\java\ $(FilePath) UTF-8

修改之前配置的 java 工具 

菜单文字：运行工具
命令：D:\Program Files\Java6\jdk1.6.0_45\bin\java.exe
参数：-classpath D:\tmp\java\. Editplus4Java D:\tmp\java\ $(FilePath)

四、QuickStart 程序的一些细节

在实际开发过程中，要及时使用 Ctrl+s 进行保存，避免异常关机时代码丢失

javac 命令在编译时会检查代码编写是否正确，所以在编译失败时可以根据返回信息判断是哪里出的问题，如下

java 命令运行程序时不需要加.class后缀

程序运行时JVM会在该类中寻找 main 方法，如果没有，会出现 java.lang.NoSuchMethodError: main，翻译一下就是没有这样一个方法错误:main

public、class、static、void为 java 关键字，{} 为编写代码的代码块，后续会详细说明

main 方法是所有 java 软件的入口方法，大到 Weblogic、Tomcat 这样的 web 服务器，小到我们的 QuickStart，都是从 main 方法入口启动程序的。编写格式必须是这样的格式 

public static void main(String[] args){
	// 执行代码
}

System.out.println("你好，Java");代码就是让程序输出一行文字“你好，Java”，并且结尾换行

.java文件（以后我们会称之为源文件）中只能有一个 public 的类，而且该类的名称需要和 .java 文件名一致。关于 public、private、protected 等访问权限修饰关键字以后会详细讲解

五、CLASSPATH 环境变量（了解为主）

当 JVM 运行时，如果设置了 CLASSPATH 变量，那么 JVM 会到该目录下寻找 java 类文件

该变量配置的方式不同，寻找顺序也不同

在我的 D:\tmp\java\quickstart\ 和 C:\ 下都有一个 QuickStart.class 文件

第一个输出：“你好，Java”

第二个输出：“你好，Java，我在C盘下”

打开 cmd ，切换到 D:\tmp\java\quickstart\ ，我们的例子始终在此目录下进行


可以使用 set 命令设置临时环境变量 

D:\>cd tmp\java\quickstart

D:\tmp\java\quickstart>set classpath			## 查看 classpath 环境变量的值
环境变量 classpath 没有定义

D:\tmp\java\quickstart>set classpath=C:\		## 设置 classpath 环境变量的值为 C:\

D:\tmp\java\quickstart>set classpath
classpath=C:\

D:\tmp\java\quickstart>set classpath=			## 清空 classpath 环境变量的值

D:\tmp\java\quickstart>set classpath
环境变量 classpath 没有定义
